#### 案例5.1：订单恢复-NEW/PROCESSING状态重新处理
**测试目标**：验证NEW/PROCESSING状态的订单在应用重启后重新进入处理流程
**前置条件**：
1. 数据库`t_exchange_order`中存在两笔订单：
   - 订单1：`BUY0000000000001`，状态`NEW`，创建时间`2026-02-18T09:00:00`
   - 订单2：`SEL0000000000001`，状态`PROCESSING`，创建时间`2026-02-18T09:00:01`

    ```
    {
        "clOrderId": "SEL0000000000001",
        "market": "XSHG",
        "securityId": "600030",
        "side": "B",
        "price": 20.10,
        "qty": 100,
        "shareholderId": "SH00000001"
    }

    {
        "clOrderId": "BUY0000000000001",
        "market": "XSHG",
        "securityId": "600030",
        "side": "B",
        "price": 20.20,
        "qty": 200,
        "shareholderId": "SH00000002"
    }
    ```

1. 应用已停止，订单簿中无上述订单

**输入数据**：
- 操作：重启交易撮合系统

**预期输出**：
- 日志输出：
  ```
  【订单恢复】需要恢复的状态：新建订单,处理中
  【订单恢复】订单[BUY0000000000001]（新建订单）重新处理
  【订单恢复】订单[SEL0000000000001]（处理中）重新处理
  ```
- 数据库`t_exchange_order`中：
  - 订单1状态更新为`NOT_FILLED`/`PART_FILLED`/`FULL_FILLED`（根据撮合结果）
  - 订单2状态更新为`NOT_FILLED`/`PART_FILLED`/`FULL_FILLED`（根据撮合结果）
- 订单簿中：
  - 若未撮合成功，订单1和订单2加入对应方向的订单簿

    ```
    原数据库：
    3	BUY0000000000001	SH00000002	XSHG	600030	B	200	200	20.20	2	2026-02-20 01:34:04	1
    1	SEL0000000000001	SH00000001	XSHG	600030	S	100	100	20.10	0	2026-02-20 01:27:26	1

    Recover后数据库：(撮合成功)
    3	BUY0000000000001	SH00000002	XSHG	600030	B	100	200	20.20	4	2026-02-20 01:34:04	3
    1	SEL0000000000001	SH00000001	XSHG	600030	S	0	100	20.10	5	2026-02-20 01:27:26	3

    trade数据库增加：
    1	527114527F28	BUY0000000000001	SEL0000000000001	100	20.15	2026-02-20 02:51:55	XSHG	600030	SH00000002	SH00000001
    ```

#### 案例5.2：订单恢复-FULL_FILLED/REJECTED/RISK_REJECT终态跳过
**测试目标**：验证FULL_FILLED/REJECTED/RISK_REJECT终态的订单在应用重启后被跳过，不加入OrderBook
**前置条件**：
1. 数据库`t_exchange_order`中存在两笔订单：
    ```
    数据库中数据：
    3	BUY0000000000001	SH00000002	XSHG	600030	B	100	200	20.20	6	2026-02-20 01:34:04	3
    1	SEL0000000000001	SH00000001	XSHG	600030	S	0	100	20.10	5	2026-02-20 01:27:26	3
    ```
2. 订单簿中无上述订单
3. 应用已停止


**预期输出**：
- 日志输出：
  ```
    2026-02-20 02:57:50.346 [main] INFO  c.e.t.i.persistence.OrderRecoveryService - 【订单恢复】完成，总计待恢复0笔，成功0笔，跳过0笔，失败0笔
    2026-02-20 02:57:50.347 [main] INFO  c.e.t.i.persistence.OrderRecoveryService - 【订单恢复】数据加载完成，涉及股票[]，开始执行全局主动撮合计
  ```
